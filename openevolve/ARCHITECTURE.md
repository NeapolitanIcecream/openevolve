# OpenEvolve 架构

本文档描述了 OpenEvolve 系统的顶层架构、功能和接口设计。

## 1. 架构

OpenEvolve 采用基于大型语言模型（LLM）的进化算法框架。其核心架构由以下几个主要组件构成：

- **控制器 (`controller.py`)**: 作为系统的核心协调器，负责驱动整个进化循环。它从程序数据库中选择种子程序，生成提示，调用 LLM 生成新的程序，然后对新程序进行评估，并将结果存回数据库。
- **命令行界面 (`cli.py`)**: 系统的用户入口，负责解析用户输入，初始化配置，并启动控制器开始进化过程。
- **配置模块 (`config.py`)**: 管理系统的所有配置参数，例如 LLM 设置、数据库路径、评估参数等。
- **数据库 (`database.py`)**: 持久化存储所有生成的程序、它们的变异历史、评估分数和其它元数据。这是实现进化过程状态记忆的关键。
- **评估器 (`evaluator.py`)**: 负责执行和评估一个程序的功能和性能。它通过加载并执行用户提供的评估脚本来工作。为了确保安全性和一致性，**评估脚本自身需要负责实现沙箱环境**。评估器会根据预定义的指标返回评估结果 (`evaluation_result.py`)。此外，评估器还可以利用 LLM 对代码质量进行辅助评估。
- **大型语言模型 (`llm/` 目录)**: 封装了与不同大型语言模型（如 Gemini, GPT 等）交互的客户端逻辑。控制器通过此模块来生成新的程序代码。
- **提示工程 (`prompt/` 目录)**: 负责构建和管理用于指导 LLM 生成代码的提示（Prompt）。
- **工具集 (`utils/` 目录)**: 提供项目范围内的通用辅助函数。

## 2. 功能

本系统的主要功能是通过进化算法，自动发现和优化算法。

1.  **初始化**: 根据用户配置，创建或加载一个程序数据库。
2.  **进化循环**:
    - **选择 (Selection)**: 控制器从数据库中选择一个或多个高质量的程序作为"父代"。
    - **变异 (Mutation)**: 控制器基于父代程序，利用 `prompt` 模块生成一个"变异提示"，并调用 `llm` 模块来生成新的"子代"程序。
    - **评估 (Evaluation)**: `evaluator` 模块执行子代程序，并根据其正确性、性能或其他指标给出评分。
    - **存储 (Storage)**: 新生成的程序及其评估结果被存储到 `database` 中，以备下一轮选择。
3.  **可持续性**: 整个过程可以中断和恢复，因为所有状态都保存在数据库中。

## 3. 接口设计

- **内部接口**: 各组件间的交互是松耦合的。控制器通过定义好的类和方法调用评估器、数据库和LLM模块。数据通过特定的数据结构（如 `EvaluationResult`）进行传递。
- **外部接口**:
    - 用户通过 `cli.py` 与系统交互，可以指定配置文件、启动、停止和监控进化过程。
    - `config.py` 文件定义了用户可以调整的几乎所有参数，为用户提供了高度的灵活性。
- **扩展性接口**:
    - **LLM**: 可以通过在 `llm` 目录中添加新的客户端轻松支持新的大语言模型。
    - **评估器**: 可以通过实现 `evaluator` 的基类来为新的问题领域定制评估逻辑。
