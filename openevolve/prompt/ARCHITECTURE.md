# Prompt 模块架构

本文档描述了 `prompt` 模块的架构、功能和接口设计。

## 1. 架构

`prompt` 模块的核心职责是构建高质量的提示（Prompts），用以引导大型语言模型（LLM）生成期望的、有创造性的代码变体。

- **`templates.py`**: 该文件存储了用于生成提示的字符串模板，并且实现了一个 `TemplateManager` 类。这个类负责加载、管理和获取模板，支持从默认配置或外部文件目录加载模板。
- **`sampler.py`**: 这是模块的核心逻辑所在。`PromptSampler` 类负责动态地构建一个完整的提示。它会接收来自调用方（如 `controller`）的一个或多个"父代"程序及其上下文信息，然后使用 `TemplateManager` 将这些信息填充到模板里，最终生成一个结构化的提示，供 `llm` 模块使用。
- **`__init__.py`**: 将 `PromptSampler` 类暴露出来，供 `controller` 等上层模块调用。

## 2. 功能

- **提示生成**: 根据输入的父代程序和指定的变异策略，生成一个完整的、可以直接发送给 LLM 的提示。
- **上下文注入**: 能够将丰富的上下文信息（例如，问题描述、表现良好的历史代码、要避免的模式等）注入到提示中，以提高 LLM 生成代码的质量和相关性。
- **策略化**: `PromptSampler` 的设计允许实现不同的采样和提示构建策略。例如，可以选择一个父代进行小幅变异，或选择两个父代进行"杂交"，这都体现在最终生成的提示内容上。

## 3. 接口设计

- **输入**: `PromptSampler` 的核心方法（例如 `build_prompt`）接收来自 `controller` 的参数，这些参数通常包括：
    - 一个或多个父代程序对象。
    - 程序的历史和上下文信息（例如，历史指标、相关的代码片段等）。
- **输出**: 该方法返回一个包含 `system` 和 `user` 键的字典，代表准备就绪的提示。
- **与外部模块的交互**:
    - `controller` 创建并调用 `PromptSampler` 实例来为每一次的进化（变异）生成提示。
    - `PromptSampler` 自身不直接与 `database` 交互，而是依赖 `controller` 传入所需的数据。
