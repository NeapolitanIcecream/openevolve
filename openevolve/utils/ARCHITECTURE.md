# Utils 模块架构

本文档描述了 `utils` 模块的架构、功能和接口设计。

## 1. 架构

`utils` 模块是一个通用的辅助函数库，提供了在整个 OpenEvolve 项目中被广泛复用的功能。它被组织为多个按功能划分的文件，以保持代码的清晰和模块化。

- **`async_utils.py`**: 包含与异步编程相关的辅助函数。一个关键功能是提供了并行执行多个异步任务并收集其结果的机制，这对于 `llm.Ensemble` 模型同时调用多个 LLM 服务至关重要。
- **`code_utils.py`**: 提供了处理和分析源代码的工具。这包括从 LLM 返回的文本中提取代码块、解析和应用自定义的 `diff` 格式、以及计算代码的编辑距离。
- **`format_utils.py`**: 包含用于格式化字符串和数据的函数，特别是将评估指标字典转换为人类可读的字符串。
- **`metrics_utils.py`**: 提供用于处理评估指标的函数，例如安全地计算指标字典中数值型数据的总和或平均值。

## 2. 功能

- **提供通用能力**: 该模块的核心功能是避免在项目的不同部分重复造轮子，将通用的、底层的操作（如异步执行、代码解析）封装成可复用的函数。
- **解耦**: 通过将这些辅助功能集中到 `utils` 模块，其它业务逻辑模块（如 `controller`, `evaluator`）可以更专注于自身的核心职责，而无需关心这些底层细节的实现。
- **代码处理**: 在与 LLM 交互时，经常需要对模型生成的原始文本进行清理和解析，`code_utils` 在其中扮演了重要角色，负责提取代码、应用变更等。

## 3. 接口设计

- **接口形式**: `utils` 模块主要以函数集合的形式提供服务。这些函数通常是无状态的，接收特定的输入并返回可预测的输出。
- **调用方**: 项目中的几乎所有其他模块都可能根据需要调用 `utils` 中的函数。
    - `controller` 和 `llm` 模块可能会使用 `async_utils` 来实现并行化。
    - `evaluator` 和 `controller` 可能会使用 `code_utils` 来处理 LLM 生成的代码。
    - 任何需要生成日志或格式化输出的模块都可能用到 `format_utils`。
- **扩展性**: 当出现新的通用需求时，可以通过向此模块添加新的函数或新的文件来轻松地进行扩展。
